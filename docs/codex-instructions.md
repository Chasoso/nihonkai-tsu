## Nihonkai Tsu

---

# プロジェクト背景

このリポジトリには、すでにプロトタイプが存在します。

現在のサイトは **GitHub Pages** で公開されています。

今回の目的は、

**既存機能を壊さずにUIを改善すること**

です。

---

# 重要ルール

Codexは次のルールを守ること。

1. プロジェクトを作り直さない
2. 既存構造を可能な限り活用する
3. 不要なライブラリを追加しない
4. GitHub Pages互換を維持する
5. 段階的に改善する

---

# 開発ステップ

開発は以下の順で進める。

Step1  
リポジトリ構造を確認する

Step2  
UIレイアウトを改善する

Step3  
魚詳細と共有UIを改善する

Step4  
バッヂ表示を改善する

# デザイン目標

UIは次の印象を目指す。

- モダン
- ミニマル
- クリーン
- プレミアム

---

# 実装方法

変更を実装する前に：

1. リポジトリを調査する
2. UI関連ファイルを特定する
3. 改修方針を提案する

いきなり実装を始めないこと。

---

# 期待される結果

改修は次を満たす必要がある。

- 視覚的品質が向上している
- 既存機能が壊れていない
- コードが整理されている
